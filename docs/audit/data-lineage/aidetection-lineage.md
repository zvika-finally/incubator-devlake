# AI Detection Data Lineage

## Overview

The AI Detection dashboard identifies AI-assisted code contributions and measures their impact on code quality and team productivity. It uses a multi-signal approach combining explicit markers (Co-authored-by tags) with behavioral patterns (commit velocity, PR characteristics).

## Table Schemas

### ai_usage_signals

| Column | Type | Source | Transformation |
|--------|------|--------|----------------|
| id | varchar(255) | Generated | `{pull_request_id}` |
| pull_request_id | varchar(255) | pull_requests.id | Direct |
| project_name | varchar(255) | project_mapping | Via PR → repo → project |
| ai_confidence_score | int | scoreAIConfidence | Weighted 0-100 |
| detected_tool | varchar(50) | detectExplicitSignals | Copilot/Cursor/Claude/etc |
| explicit_tool_detected | bool | detectExplicitSignals | True if explicit marker found |
| explicit_tools | text | detectExplicitSignals | Comma-separated detected tools |
| explicit_signal_score | int | detectExplicitSignals | 0-70 points |
| rapid_commit_score | int | analyzeCommitPatterns | 0-30 points |
| pr_size_score | int | analyzePRCharacteristics | 0-20 points |
| lines_per_minute_score | int | analyzePRCharacteristics | 0-25 points |
| duplication_score | int | scoreAIConfidence | 0-15 points (reserved, currently 0) |
| generic_message_score | int | analyzeCommitPatterns | 0-10 points |
| velocity_multiplier | decimal(3,2) | analyzePRCharacteristics | PR cycle vs baseline |
| detected_at | timestamp | Generated | Detection timestamp |

## Confidence Score Calculation

```
ai_confidence_score = MIN(100,
    explicit_signal_score    (0-70, highest weight if present)
  + rapid_commit_score       (0-30)
  + pr_size_score            (0-20)
  + lines_per_minute_score   (0-25)
  + duplication_score        (0-15)
  + generic_message_score    (0-10)
)

Maximum: 100 points
```

### Explicit Signal Detection

Searches PR titles, bodies, and commit messages for:
- `Co-authored-by: *[bot]` patterns
- `Co-authored-by: *copilot*`, `*cursor*`, `*claude*`, `*aider*`
- Keywords: "generated by", "AI-generated", "copilot", "cursor", "claude code"

## Confidence Thresholds

| Level | Score Range | Dashboard Color | Interpretation |
|-------|-------------|-----------------|----------------|
| High | >= 70 | Green | Strong AI indicators |
| Medium | 40-69 | Orange | Possible AI assistance |
| Low | < 40 | Red | Likely human-only |
| Explicit | Any + explicit_tool_detected | Blue badge | Confirmed AI tool |

Effective detection threshold for classification tasks:
- `options.confidenceThreshold` if provided
- else `settings.detectionThreshold` if provided
- else fallback `65`

## Infrastructure File Exclusions (Churn Analysis)

The churn analysis excludes these patterns from calculations to avoid noise:
- `.github/**` - GitHub workflows and configs
- `package.json`, `package-lock.json`, `yarn.lock` - Node dependencies
- `go.mod`, `go.sum` - Go dependencies
- `*.config.*`, `*.conf` - Configuration files
- `Makefile`, `Dockerfile`, `docker-compose*` - Build files
- `*.md` - Documentation files

## Plugin Task Execution Order

```
aidetector plugin subtasks (from impl.go):
  1. detectExplicitSignals      → Scan PR/commit text for AI markers
  2. analyzeCommitPatterns      → Detect rapid commits, generic message patterns
  3. analyzePRCharacteristics   → Calculate PR metrics vs baseline
  4. scoreAIConfidence          → Combine scores → ai_usage_signals
  5. calculateAIImpact          → Before/after comparison
  6. analyzeCodeChurn           → Track post-merge modifications
```

## Dependencies

| Dependency | Required Tables | Notes |
|------------|-----------------|-------|
| GitHub/GitLab plugin | pull_requests, commits, commit_files, pull_request_commits | Must run before aidetector |
| project_mapping | project_mapping | Links repos to projects |
| DORA plugin | project_pr_metrics | Optional: provides lead_time for impact metrics |

## Data Freshness

- **ai_usage_signals**: Updated on each pipeline run, `detected_at` tracks last calculation
- **developer_baselines**: Recalculated on each run using rolling window
- **ai_churn_metrics**: Requires 30 days post-merge to be complete
- **project_churn_summaries**: Aggregated from ai_churn_metrics

## Known Limitations

1. **Explicit detection bias**: PRs with AI markers are definitively flagged, but many AI-assisted PRs lack markers
2. **Baseline period**: New developers may have incomplete baselines
3. **Churn window**: PRs merged within last 30 days have incomplete churn data
4. **External API data**: Cursor/Claude Code metrics require separate API integration (optional)
